# Render Fix: Switch from Web Service to Static Site

## Problem

A previous setup used Render Web Service configuration for a frontend-only app.

## Correct setup for KrayoNova

KrayoNova should run as a static deployment with Next.js export output.

### Required Render configuration

- Service Type: `Static Site`
- Build Command: `npm install && npm run build`
- Publish Directory: `dist`
- Rewrite rule: `/* -> /index.html` (type `Rewrite`)

## Why this fixes deployment issues

- Removes dependency on Node runtime start commands.
- Uses static files generated by Next.js export.
- Supports deep-link refreshes via rewrite fallback.

## Quick verification checklist

- `render.yaml` uses `type: static_site`
- No start command is configured
- `next.config.ts` includes `output: "export"`
- Build generates `dist/`
- Routes load correctly after refresh
